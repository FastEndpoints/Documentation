# Operations

## Deploy model
Static site generated by `npm run build` into `docs/`. Hosted as a static docs site (Cloudflare Pages style preview host referenced in README). No in-repo CI/CD config (no `.github/` workflows).

| Environment | URL (documented) |
| --- | --- |
| Production | `https://fast-endpoints.com` |
| Dev preview | `https://dev.fastendpoints-doc-site.pages.dev` |
| Alternate Pages URL in config | `https://fastendpoints-doc-site.pages.dev` |

`src/config.ts` currently hardcodes `isProduction = true`, so `siteUrl` and related SEO fields use the production host even when building non-prod unless that flag is changed.

## Services and ports
- Dev: Vite default dev server (local only).
- No app-owned databases, queues, or long-running jobs.
- Build-time outbound: GitHub API for contributors.

## Data stores
None. Artifacts: prerendered HTML, `search.json`, `sitemap.xml`, static assets.

## Config and observability
Config lives in source (`src/config.ts`), not env files:
- Site SEO / Open Graph fields
- `github`, `discord` links
- `algolia.apiKey` / `algolia.appId` keys exist in config (client search implementation uses FlexSearch + `/search.json`; treat Algolia fields as config surface, not values to copy into OKF)
- Navbar links in `navbar` export

No first-party APM/logging stack in-repo. Rely on host platform logs if any.

## Caveats
- `/docs` output directory is gitignored; publish pipeline must build before deploy.
- `handleMissingId: 'fail'` makes bad anchors a deploy-blocking build failure.
- Sitemap URLs use `config.siteUrl`; wrong production flag skews absolute URLs.
